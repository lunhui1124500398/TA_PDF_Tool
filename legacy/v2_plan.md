# 第二版 AI 自动批改拆解

目标不是一口气把“OCR + 判题 + 批语”做成黑盒，而是拆成一串可以逐步验收的步骤。建议顺序如下。

## 阶段 0：先把第一版数据流稳定下来

输入统一成下面这套结构：

- 每个学生一个规范文件名：`学号-姓名.pdf`
- 一份合并后的 PDF 供人工批改
- 一份 `manifest.json` 记录每个学生的页码范围、批语区位置、批语内容

这一步已经能直接服务当前助教工作，也会给第二版提供稳定输入。

## 阶段 1：OCR 基线验证

目标：

- 跑通 PaddleOCR 对作业 PDF 的识别
- 把每页识别结果落成结构化 JSON

建议最小验证样本：

- 先挑 5 份学生作业
- 覆盖打印体、手写体、扫描件、拍照件几种情况

验收标准：

- 页级 OCR 成功率可接受
- 中文、公式编号、题号至少能大致识别
- 能从 OCR 结果中追溯回原始页码

输出示例：

```json
{
  "student_id": "2500011895",
  "pages": [
    {
      "page_index": 1,
      "ocr_text": "...",
      "blocks": []
    }
  ]
}
```

## 阶段 2：答案标准结构化

目标：

- 把标准答案从 PDF / Word / Markdown 变成机器可用的题目级结构

建议先做成人工维护的 JSON：

```json
{
  "assignment": "HW4&5",
  "questions": [
    {
      "question_id": "Q1",
      "prompt": "题目内容摘要",
      "reference_answer": "标准答案",
      "scoring_rubric": [
        "关键公式正确",
        "推导过程完整",
        "最终结论正确"
      ],
      "max_score": 10
    }
  ]
}
```

验收标准：

- 至少能完整表达每道题的参考答案和评分点
- 后续 AI 不需要再去猜“这一题看什么”

## 阶段 3：题目切分

目标：

- 把 OCR 文本按题号切成 `Q1 / Q2 / Q3`

实现建议：

- 先走规则法：正则匹配题号、阿拉伯数字、括号编号
- 对切分失败的页标成 `low_confidence`

验收标准：

- 绝大多数常规题号能切出来
- 每段学生答案都能映射到具体题目

## 阶段 4：AI 题目级判分

输入：

- 单个学生某道题的 OCR 结果
- 对应题目的标准答案与评分点

输出：

```json
{
  "question_id": "Q1",
  "score": 8,
  "max_score": 10,
  "verdict": "部分正确",
  "feedback": "关键思路正确，但最后一步代入有误。",
  "confidence": 0.71
}
```

关键设计：

- 先让 AI 给“评分依据 + 置信度”，不要只给分数
- 低置信度题目进入人工复核
- 所有判分结果都要可追溯到 OCR 片段和标准答案

验收标准：

- 在 10 到 20 份样本上，人和 AI 的评分差异可分析
- 明显错误题能稳定指出原因

## 阶段 5：总评与批语生成

目标：

- 汇总每题判分结果，生成总评和建议批语

建议输出两层：

1. 结构化结果：总分、分题得分、扣分点
2. 自然语言批语：适合直接放在第一页批语区

验收标准：

- 批语不空泛
- 能反映真实扣分点
- 不和分题结果冲突

## 阶段 6：写回 PDF 与人工复核工作台

目标：

- 将 AI 批语和分题摘要写回 PDF
- 人工能快速修改 AI 结果，而不是被 AI 绑死

建议：

- 第一版先只写总评到第一页批语区
- 分题细节保存到 JSON / CSV
- 后续再考虑把分题批注定位到具体页边

## 阶段 7：评估与上线

上线前至少做三类评估：

1. OCR 质量评估
2. 题目切分准确率评估
3. AI 判分与人工评分一致性评估

建议先采用“AI 预批 + 人工复核”的模式，不要直接全自动放分。

## 推荐的最小可行版本

如果要尽快进入第二版开发，我建议先做这一条最短链路：

1. 选 5 份作业样本
2. 用 PaddleOCR 跑出页级文字
3. 手工准备一份标准答案 JSON
4. 只做 1 到 2 道题的切分与 AI 判分
5. 输出结构化结果和一段总评
6. 最后把总评写回第一页批语区

这样每一步都能单独看结果，也更容易定位问题出在 OCR、切题还是判分上。
